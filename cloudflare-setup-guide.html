Ship Quiltly on Cloudflare Workers

This project is a Nuxt 4 + NuxtHub app. The deployment path is Cloudflare Workers with NuxtHub-managed database, blob storage, cache, auth tables, and the existing Cloudflare Email binding in wrangler.jsonc.

Deployment Flow

Move left to right. Do not start with Pages-only settings, because this app has server routes, auth, blob uploads, and database migrations.

1

Local build is clean

Install with pnpm, generate Nuxt types, lint, typecheck, build, and confirm Nitro reports the cloudflare-module preset.

2

Cloudflare account is ready

Wrangler must be logged in to the account that will own the Worker and bindings.

3

Bindings exist

NuxtHub covers db, blob, and cache. The committed Wrangler file adds Email Sending and observability.

4

Deploy and test

Deploy with Wrangler, create a test account, upload an image, save a bookmark, and check logs.

Commands

Run these from the repository root. This project already declares pnpm@11.8.0 in package.json.

Local

Prepare and verify

Use this before touching Cloudflare so deployment errors are not mixed with local errors.

pnpm install
pnpm run lint
pnpm run typecheck
pnpm run build:cloudflare
Auth

Log in to Cloudflare

Wrangler must show the right account before creating resources or deploying.

pnpm exec wrangler login
pnpm exec wrangler whoami
Secrets

Add production secrets

Keep secrets in Cloudflare, not in git. Use the dashboard or Wrangler secret commands.

pnpm exec wrangler secret put NUXT_BETTER_AUTH_SECRET
pnpm exec wrangler secret put NUXT_PUBLIC_SITE_URL
pnpm exec wrangler secret put NUXT_CF_ACCOUNT_ID
pnpm exec wrangler secret put NUXT_CF_EMAIL_API_TOKEN
Deploy

Publish the Worker

The app now sets nitro.preset: 'cloudflare-module'. Nitro generates the deployable Wrangler config in .output.

pnpm run deploy:cloudflare

Cloudflare Console

Use this as the dashboard checklist after the first deploy has created or linked the Worker.

Worker

Workers & Pages -> quiltly -> Settings. Confirm compatibility flag nodejs_compat and observability logs are enabled.

Data

Confirm NuxtHub resources are present for SQLite/D1, blob/R2, and cache/KV. Migrations should create auth, projects, supplies, bookmarks, and project supplies tables.

Email

Enable Cloudflare Email Sending or route-compatible sending for noreply@quiltly.app. Auth emails use the REST credentials listed in the secrets step and now fail loudly if production mail cannot be sent.

Post-Deploy Smoke Test

These checks exercise the parts most likely to fail from missing bindings, migrations, or secrets.

In the app

  • Create a new user and verify the email flow.
  • Add one supply and one project.
  • Link the supply to the project, then mark the project completed.
  • Upload an image and confirm it renders from /api/files/....
  • Save a bookmark and confirm metadata is fetched.

From Wrangler

Tail logs while running the app smoke test. Email, D1, and blob failures should show up here.

pnpm exec wrangler tail quiltly

If this command differs for your Wrangler version, use the dashboard log stream for the Worker.

Required Config Check

The current app must build for Cloudflare before Wrangler deploys it. This is the one setup step to verify carefully.

Nitro target

nuxt.config.ts now sets nitro.preset to cloudflare-module. A successful Cloudflare-targeted build should report Nitro preset: cloudflare-module and should not include local platform binaries such as Darwin-only image tooling.

Project-specific notes

The current wrangler.jsonc commits the Worker name, nodejs_compat, Email binding, and observability. Keep NuxtHub storage configuration in nuxt.config.ts, and keep NUXT_BETTER_AUTH_SECRET, NUXT_PUBLIC_SITE_URL, NUXT_CF_ACCOUNT_ID, and NUXT_CF_EMAIL_API_TOKEN configured as Cloudflare secrets. Build with pnpm run build:cloudflare so the production URL is not inherited from the local .env file.
